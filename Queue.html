Queue Algorithm

Queue Algorithm

Store a list of items in which an item can be inserted at one end and removed from the other end only.
  

FIFO Principle of Queue :-

Queue follows FIFO (fast in, first out) order or approach in which the operations are performed. This means one element is inserted first which is to be deleted first.

Queue Operations :-

    enqueue(x) :- Add an element to the back of a queue.
    dequeue(z) :-Remove an element from the front of a queue.
    peek() :- It returns the value of the front element of the queue.
    empty() :- Queue is used to check the associated queue container is empty or not.

 Example :-

  

Process :-

  

Step 1 :-

We initially have the element Y and Z. Then front element Z and back element Y. Add an element X to the back of a queue. After enqueue(x) operation front element Z and back element X.   

Step 2 :-

We have then performed a dequeue(z) operation, removing the element from the front of a queue. The element X is dequeue from the queue.   

Step 3 :-

When the dequeue(z) operation is performed then the peek() element will be changed. Because the front value will be changed and it returns the value of the front element of the queue.   

Step 4 :-

Finally, Queue is used to check if the associated queue container is empty or not.
