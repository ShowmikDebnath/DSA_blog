Coin Change Problem

The coin change problem is a classic problem in computer science that deals with finding the minimum number of coins required to make a given amount of change. This problem is used in various domains like finance, gaming, and cryptography. In this blog, we will explore the coin change problem in detail and understand how to solve it using various algorithms.

Problem Statement:

The problem statement is quite simple. Given an amount and a set of coins, we need to find the minimum number of coins required to make the given amount. For example, consider the following scenario:

Amount = 11
Coins = [1, 2, 5] 
            11 x 1
            5 x 2 + 1 x 1
            2 x 3 + 1 x 5
In the first case, we are using 11 coins, in the second case, we are using 3 coins, and in the third case, we are using 8 coins. Hence, the minimum number of coins required to make an amount of 11 using the coins [1, 2, 5] is 3.

Brute Force


The brute force approach to solve this problem is to try all possible combinations of coins and find the one with the minimum number of coins. This approach has exponential time complexity and is not feasible for large inputs.
